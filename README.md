## 自己PR
**業務を早期にキャッチアップできる**

接客業からエンジニアにキャリアチェンジした際には現場に1人で常駐することになりましたが、現場の方々と親交を深め、積極的に質問や発言をすることで業務フローを理解することで早期に業務にキャッチアップすること ができました。その結果、配属から2ヶ月ほどで前任者の保守業務のほとんど巻き取ることができました。

**問題解決能力が高く業務を円滑に進められる**

保守運用業務を主に担当していた際に、いかに業務を効率化し属人化を防ぐかを重要視してきました。私が現場に派遣されたときには保守業務のマニュアルがほとんどなく、前任の担当者にその都度確認をとって業務を進めていましたが、日に5〜10個の保守案件が存在していたので今後、新しい人が入ってきた際に同じようなフローで保守業務を進めていくと非効率だと考え、担当している保守業務のマニュアルを作成し、効率的に業務を進めるようにしました。

## エンジニアを目指す理由
ソーシャルゲームやSNSに触れていくうちに、サービスやアプリの仕組みについて興味が湧くと同時にそ の仕組みを作っているエンジニアの仕事に興味を持つようになりました。 また、義務教育でのプログラミングが必修化やAI技術の発展などでさらなる成長が見込まれるIT業界で 働くことで技術を身に付け、業界の成長とともに自身も成長していきたいと考え、実際にプログラミングに 触れていくうちに、問題に直面したときの大変さや辛さを経験すると同時に問題を解決したときの楽しさ や嬉しさを知り、さらに技術力を高めていきたいと考えるようになり、エンジニアを志しました。
 
## 職務要約
高校卒業後、官公庁などの大型施設の空調設備の保守運用業務に従事しました。

退職後しばらくフリーターとして生活していましたが、高校時代の恩師からの紹介で川崎工科高校に臨時的任用職員として着任。電気工事実習や情報の授業を担当。契約満了と同時に更新はせずに退職し、フリーターに戻りました。

1年間ほど飲食店のデリバリーのアルバイトをしていましたが、このままではいけないと思い直し、努力次第で大きなリターンが得られる可能性の高いエンジニアを目指すことを決めました。

プログラミングスクールで3ヶ月ほどJavaの基礎を学び、株式会社さくらコミュニュケーションに内定をいただき、1年ほど不動産関連の企業で社内システムの保守・運用業務を担当していました。
ユーザーからの意見を集めて、新規機能を追加する経験や多部署の業務を効率化するためのツールの導入の提案やヘルプデスクとしての業務なども経験させていただきました。ベンダーとのやりとりなどが主な仕事でコードを書くことがほとんどなく、開発ができる環境に行きたいと思い、退職しました。

退職後はピザチェーン店でアルバイトをしながらプログラミングの学習をしており、主にRuby・PHP・JavaScriptを学んでいます。それぞれの基礎を学んだ後にフレームワークとしてRuby on Rails・Laravel・Vue.jsを学習しました。

インフラ面の学習も同時に進めており、ポートフォリオ公開する際に、Docker・AWS・circleCIを利用して公開しています。
